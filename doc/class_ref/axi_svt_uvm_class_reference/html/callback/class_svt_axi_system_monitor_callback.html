 virtual function void
 svt_axi_system_monitor_callback::get_dynamic_coherent_to_snoop_xact_type_match
 (  svt_axi_system_monitor system_monitor , svt_axi_transaction coherent_xact , svt_axi_system_transaction sys_xact , svt_axi_snoop_transaction snoop_xact , ref bit is_coherent_to_snoop_type_match  ) 

Called whenever system monitor needs to match the type of snoop transaction to coherent transaction.

coherent_xact - A reference to the coherent data descriptor object of interest.

sys_xact - A reference to the system transaction descriptor object of interest.

snoop_xact - A reference to the snoop transaction which is being evaluated to associate with coherent_xact.

is_coherent_to_snoop_type_match - Indicates if the type of the snoop transaction corresponds to that of the coherent transaction. This variable has a default value based on the following: If set_snoop_type_for_coherent_transaction has been called on sys_xact, then the default value is sys_xact.dynamic_snoop_type If set_snoop_type_for_coherent_transaction has not been called on sys_xact, then the default value is based on the static mapping provided through SVT_AXI_RECOMMENDED_SNOOP_XACT (if svt_axi_system_configuration :: use_recommended_coherent_to_snoop_map is set) or SVT_AXI_LEGAL_SNOOP_MAPPING(if svt_axi_system_configuration :: use_recommended_coherent_to_snoop_map is not set)

Called after system monitor finds one or more master transactions that can potentially be associated with a slave transaction.

original_sys_xacts - A reference to all the master transactions that are eligible for association.

slave_xact - A reference to the slave transaction.

user_sys_xacts - A subset of original_sys_xacts that the system monitor should consider for master slave association, this is expected to be populated by the user in case the drop bit is set to 1.

drop - A bit that indicates that the user has populated user_sys_xacts and that the system monitor should consider user_sys_xacts rather than original_sys_xacts for master slave association.

 virtual function void
 svt_axi_system_monitor_callback::readonce_xact_with_overlapped_writeback_writeclean_xacts_at_eos
 (  svt_axi_system_monitor system_monitor , svt_axi_system_transaction sys_xact , ref bit bypass_check  ) 

Called after the system monitor detects WRITEBACK/WRITECLEAN xacts that overlap in time and address with a READONCE transaction and where the data in WRITEBACK/WRITECLEAN dose not match with READONCE. This callback is called at end of simulation.

Overriding implementations in extended classes must ensure that the callbacks get executed correctly.

This callback can be used to bypass the check data_integrity_with_outstanding_coherent_write_check, based on contents of sys_xact. The variable sys_xact.last_coherent_write_to_addr holds the list of WRITEBACK/WRITECLEAN transactions overlapping with READONCE transaction. The READONCE transaction can be referenced via sys_xact.master_xact

sys_xact - A reference to the queue of system transaction descriptor object which was detected as READONCE and WRITEBACK/WRITECLEAN transactions.

bypass_check - user to program this bit to 1 to bypass particular WRITEBACK/WRITECLEAN xatc data with READONCE xact data mismatch check.

Called after the system monitor detects WRITEBACK/WRITECLEAN xacts that overlap in time and address with a READONCE transaction and where the data in WRITEBACK/WRITECLEAN may match or may not match with READONCE.

The READONCE transaction can be referenced via sys_xact.master_xact

sys_xact - A reference to the queue of system transaction descriptor object which was detected as READONCE and WRITEBACK/WRITECLEAN transactions.
